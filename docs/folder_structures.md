## Database Schema Overview

| Model           | Status    | Key Fields                                                                                        |
| --------------- | --------- | -------------------------------------------------------------------------------------------------- |
| `User`          | ✅ Active | email, password, fullName, phone, role (ADMIN/MANAGER/USER)                                        |
| `Vehicle`       | ✅ Active | plateNumber (unique), vehicleType (CAR/MOTORBIKE), color, → User                                   |
| `ParkingLot`    | ✅ Active | name, address, totalSlots, lotType, zones (JSON), carHourlyRate, motorbikeHourlyRate               |
| `ParkingSlot`   | ✅ Active | zoneId, slotNumber, vehicleType, status (AVAILABLE/OCCUPIED/RESERVED/MAINTENANCE)                  |
| `Booking`       | ✅ Active | startTime, endTime, estimatedCost, status (PENDING_PAYMENT/CONFIRMED/COMPLETED/CANCELLED)          |
| `ParkingRecord` | ✅ Active | checkInTime, checkOutTime, actualCost, paymentStatus, status (CHECKED_IN/CHECKED_OUT)              |
| `Payment`       | ✅ Active | amount, method (CASH/VNPAY), status (PENDING/SUCCESS/FAILED/REFUNDED), referenceId                 |
| `MonthlyPass`   | ✅ Active | vehicleType, vehicleId (@unique), startDate, endDate, price, status (ACTIVE/EXPIRED/CANCELLED)     |
| ~~AdminLog~~    | ❌ Removed | Removed — too complex for current scope                                                            |

## Tech Stack

- **Runtime**: Node.js (ES Modules — `"type": "module"`)
- **Framework**: Express 5
- **ORM**: Prisma with PostgreSQL (NeonDB)
- **Auth**: JWT (httpOnly cookie + Bearer header)
- **Password**: bcryptjs
- **Validation**: Zod
- **Dev**: nodemon, dotenv
